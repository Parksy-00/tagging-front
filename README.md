##Stolage 디자인 문서 (미완)
#배경(Problem to solve)
	Stolage는 클라우드 저장소로 시작하지 않았다. 클라우드 저장소는 사용자가 파일을 저장된 기기에서만 접근할 수 있었던 문제를 해결하기 위해 등장한 것이다. 서비스마다 약간의 차이가 있겠지만 그 큰 틀은 모두 같다. 따라서 사용자가 그러한 서비스들에서 얻게 되는 사용자 경험에는 큰 차이가 없다. 이는 다음 두 가지 현상으로 확인할 수 있다. 첫째, 대부분의 사람들은 클라우드 서비스들을 제공 용량과 요금이라는 두 축을 위주로 비교한다. 둘째. 몇몇 사용자들은 그 제공 용량 대비 요금에 만족하지 못해, 클라우드 서비스가 제공하는 약간의 편의성을 포기한 채 NAS를 구축하기도 한다.
	Stolage는 파일을 저장하기 위해 존재하는 것이 아니고 파일을 손쉽게 관리하기 위해 존재하는 것이다. 저장기능은 부차적인 요소이다. 자신이 가진 파일을 미래에 활용하고자 하는 의지가 있는 사람이라면 누구나 그것을 적절히 관리하고자 할 것이다. 하지만 기존의 파일 시스템에서 이것은 대단히 어려운 일인데, 그 이유는 크게 세가지로 들 수 있다. 첫째, 탐색의 어려움. 둘째, 저장의 어려움, 셋째 수정의 어려움. (설명 추가 예정)

	Stolage는 파일을 관리하는 방식에 새로운 관점을 제공하여  위와 같은 문제를 해결하기 위해 시작되었다.

#탐색(Read)
	탐색은 사용자가 검색바에 원하는 파일에 대한 정보를 제공하면 수행된다. 이 정보는 반드시 이전에 파일을 저장하면서 함께 저장한 태그 중 하나여야 한다. Stolage는 태그 기반 ‘필터링’을 지원하며 임의의 단어 기반 ‘검색’은 지원하지 않는다.

	화면의 정중앙에 있는 입력바를 기준으로 하단에는 입력바에 제공한 정보를 바탕으로 찾은 파일 목록, 상단에는 사용자가 원하는 파일을 찾기 위해 필요할 법한 정보가 지금까지 입력한 정보를 바탕으로 추천된다. 사용자는 질의를 마쳤다는 사실을 알릴 필요가 없으며, 찾으려고 하는 파일을 발견하면 스스로 질의를 종료한다. 따라서 입력바 기준 상단과 하단에 존재하는 추천태그 목록과 파일 목록은 사용자가 태그의 선택과 삭제를 수행할 때마다 즉각적으로 업데이트 되어야한다. 만족스러운 사용자 경험을 위해서는 사용자가 그 속도를 ‘충분히 즉각적’ 이라고 느껴야한다.

	탐색을 완료한 상태에서 사용자는 자신이 얻은 목록의 일부, 또는 전부를 손쉽게 선택하여 다운로드, 파일명 수정, 태그 추가, 삭제, 파일 삭제 등의 행동을 할 수 있어야 한다. 사용자가 탐색에서 얻은 파일 목록의 일부에 관심을 둔 다는 것은 자신이 찾으려는 파일이 정확히 무엇인지 모르는 상태에서 그것을 묘사하여, 다시 말해 적절한 정보를 순차적으로 제공하여 파일을 찾는 것이고, 파일 목록의 전부에 관심을 둔다는 것은 자신이 찾으려는 파일(들)이 정확히 무엇인지 아는 상태에서 그것들의 공통 특징들을 묘사하여 원하는 파일목록을 한번에 얻는 것이다.

어떤 행동을 위해서건, 사용자의 모든 탐색은 입력바에서 시작되어 입력바에서 끝나야하며, 그 루프가 모든 사용자에게 직관적으로 이해되어야한다.
 
#수정(Update)
	수정은 기존에 존재하던 것을 삭제하거나 변경하는 것을 의미한다. 수정의 많은 부분은 앞서 탐색 부분에서 밝혔으나 가장 중요한 이야기가 남았다. 태그의 수정과 파일의 수정은 다르다. 첫째, 태그의 수정은 입력바 상단에서 일어나며 파일의 수정은 입력바 하단에서 일어나야만 한다. 둘째, 태그의 수정은 모든 파일에 반영되며 파일의 수정은 선택한 파일에만 반영된다.


#저장(Create)
	저장은 사용자가 새로운 파일을 업로드 하면서 일어난다. 사용자는 파일을 업로드 하면서 미래에 그 파일을 탐색하기 위해 필요한 정보를 태그의 형태로 제공하게 되는데, 이는 앞서 탐색과정에서 이용한 입력바를 이용하여 이루어진다. 탐색과정과의 차이는 사용자가 아직 존재하지 않는 태그 또한 입력할 수 있다는 것이다. 파일이 업로드될 때, 입력바에 있는 모든 태그들이 그 파일의 태그가 된다.

	파일에 태그를 부여하는 작업은 분명 미래의 탐색과정을 편하게 만들어주며, 기존의 폴더구조에 비해 편리하지만, 여전히 이를 불편하게 생각할 수 있으므로, 사용자가 직접 태그를 부여하지 않아도 자동으로 추가되는 태그가 충분히 존재해야한다. 예를 들어 파일의 확장자, 업로드 날짜 등이 가능한 예시가 될 것이다. 특히 폴더를 통째로 업로드 하는 경우 상위 폴더의 이름이 하위 폴더와 파일들의 태그가 되도록 하는 것은 이미 계층 형태(폴더 구조)의 파일들을 가지고 있는 사람이 Stolage로 이동할 충분한 동기가 될 수 있으므로 반드시 구현해야한다.
